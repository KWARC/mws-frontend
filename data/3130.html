87.2-4 Functions for meeting ordered partitions

A kind of refinement that occurs in particular in the normalizer calculation involves computing the meet of Π Π \Pi (cf. lines 6ff. above) with an arbitrary other partition Λ Λ \Lambda , not just with one point. To do this efficiently, GAP uses the following two functions.

StratMeetPartition( R 𝑅 R , Π Π \Pi , Λ Λ \Lambda [, g 𝑔 g ] )

MeetPartitionStrat( R 𝑅 R , I 𝐼 I {, Λ ′ superscript Λ ′ \Lambda^{\prime} }[, { g ′ superscript 𝑔 ′ g^{\prime} }], s ⁢ t ⁢ r ⁢ a ⁢ t 𝑠 𝑡 𝑟 𝑎 𝑡 strat )

Such a StratMeetPartition command would typically appear in the function call R 𝑅 R .nextLevel ( Π , R ) Π 𝑅 (\Pi,R) (during the refinement of Π i subscript Π 𝑖 \Pi_{i} to Π i + 1 subscript Π 𝑖 1 \Pi_{{i+1}} ). This command replaces Π Π \Pi by Π ∧ Λ Π Λ \Pi\wedge\Lambda (thereby changing the second argument) and returns a "meet strategy" s ⁢ t ⁢ r ⁢ a ⁢ t 𝑠 𝑡 𝑟 𝑎 𝑡 strat . This is (for us) a black box which serves two purposes: First, it allows GAP to calculate faster the corresponding meet Σ ∧ Λ ′ Σ superscript Λ ′ \Sigma\wedge\Lambda^{\prime} , which must then appear in a Refinements function (during the refinement of Σ i subscript Σ 𝑖 \Sigma_{i} to Σ i + 1 subscript Σ 𝑖 1 \Sigma_{{i+1}} ). It is faster to compute Σ ∧ Λ ′ Σ superscript Λ ′ \Sigma\wedge\Lambda^{\prime} with the "meet strategy" of Π ∧ Λ Π Λ \Pi\wedge\Lambda because if the refinement of Σ Σ \Sigma is successful at all, the intersection of a cell from the left hand side of the ∧ \wedge sign with a cell from the right hand side must have the same size in both cases (and s ⁢ t ⁢ r ⁢ a ⁢ t 𝑠 𝑡 𝑟 𝑎 𝑡 strat records these sizes, so that only non-empty intersections must be calculated for Σ ∧ Λ ′ Σ superscript Λ ′ \Sigma\wedge\Lambda^{\prime} ). Second, if there is a discrepancy between the behaviour prescribed by s ⁢ t ⁢ r ⁢ a ⁢ t 𝑠 𝑡 𝑟 𝑎 𝑡 strat and the behaviour observed when refining Σ Σ \Sigma , the refinement can immediately be abandoned.

On the other hand, if you only want to meet a partition Π Π \Pi with Λ Λ \Lambda for a one-time use, without recording a strategy, you can simply type StratMeetPartition ( Π , Λ ) Π Λ (\Pi,\Lambda) as in the following example, which also demonstrates some other partition-related commands.

gap> P := Partition( [[1,2],[3,4,5],[6]] );;  Cells( P );
[ [ 1, 2 ], [ 3, 4, 5 ], [ 6 ] ]
gap> Q := Partition( OnTuplesTuples( last, (1,3,6) ) );;  Cells( Q );
[ [ 3, 2 ], [ 6, 4, 5 ], [ 1 ] ]
gap> StratMeetPartition( P, Q );
[  ]
gap> # The ``meet strategy'' was not recorded, ignore this result.
gap> Cells( P );
[ [ 1 ], [ 5, 4 ], [ 6 ], [ 2 ], [ 3 ] ]

You can even say StratMeetPartition ( Π , Δ ) Π Δ (\Pi,\Delta) where Δ Δ \Delta is simply a subset of Ω Ω \Omega , it will then be interpreted as the partition ( Δ , Ω ∖ Δ ) Δ Ω Δ (\Delta,\Omega\setminus\Delta) .

GAP makes use of the advantages of a "meet strategy" if the refinement function in Refinements contains a MeetPartitionStrat command where s ⁢ t ⁢ r ⁢ a ⁢ t 𝑠 𝑡 𝑟 𝑎 𝑡 strat is the "meet strategy" calculated by StratMeetPartition before. Such a command replaces I 𝐼 I .partition by its meet with Λ ′ superscript Λ ′ \Lambda^{\prime} , again changing the argument I 𝐼 I . The necessary reversal of these changes when backtracking from a node (and prescribing the next possible image for a base point) is automatically done by the function PartitionBacktrack.
