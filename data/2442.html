6.4 Break Loops

When an error has occurred or when you interrupt GAP (usually by hitting Ctrl-C) GAP enters a break loop, that is in most respects like the main read eval print loop (seeÂ 6.1). That is, you can enter statements, GAP reads them, evaluates them, and shows the result if any. However those evaluations happen within the context in which the error occurred. So you can look at the arguments and local variables of the functions that were active when the error happened and even change them. The prompt is changed from gap> to brk> to indicate that you are in a break loop.

gap> 1/0;
Rational operations: <divisor> must not be zero
not in any function
Entering break read-eval-print loop ...
you can 'quit;' to quit to outer loop, or
you can replace <divisor> via 'return <divisor>;' to continue

If errors occur within a break loop GAP enters another break loop at a deeper level. This is indicated by a number appended to brk:

brk> 1/0;
Rational operations: <divisor> must not be zero
not in any function
Entering break read-eval-print loop ...
you can 'quit;' to quit to outer loop, or
you can replace <divisor> via 'return <divisor>;' to continue
brk_02>

There are two ways to leave a break loop, see 6.4-1 and 6.4-2.
