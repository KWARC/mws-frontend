21.10 Comparisons of Lists

list1 = list2

list1 <> list2

Two lists list1 and list2 are equal if and only if for every index i 𝑖 i , either both entries list1 [ i ] delimited-[] 𝑖 [i] and list2 [ i ] delimited-[] 𝑖 [i] are unbound, or both are bound and are equal, i.e., list1 [ i ] = delimited-[] 𝑖 absent [i]= list2 [ i ] delimited-[] 𝑖 [i] is true.

gap> [ 1, 2, 3 ] = [ 1, 2, 3 ];
true
gap> [ , 2, 3 ] = [ 1, 2, ];
false
gap> [ 1, 2, 3 ] = [ 3, 2, 1 ];
false

This definition will cause problems with lists which are their own entries. Comparing two such lists for equality may lead to an infinite recursion in the kernel if the list comparison has to compare the list entries which are in fact the lists themselves, and then GAP crashes.

list1 < list2

list1 <= list2

Lists are ordered lexicographically. Unbound entries are smaller than any bound entry. That implies the following behaviour. Let i 𝑖 i be the smallest positive integer i 𝑖 i such that list1 and list2 at position i 𝑖 i differ, i.e., either exactly one of list1 [ i ] delimited-[] 𝑖 [i] , list2 [ i ] delimited-[] 𝑖 [i] is bound or both entries are bound and differ. Then list1 is less than list2 if either list1 [ i ] delimited-[] 𝑖 [i] is unbound (and list2 [ i ] delimited-[] 𝑖 [i] is not) or both are bound and list1 [ i ] delimited-[] 𝑖 [i] < list2 [ i ] delimited-[] 𝑖 [i] is true.

gap> [ 1, 2, 3, 4 ] < [ 1, 2, 4, 8 ]; # <list1>[3] < <list2>[3]
true
gap> [ 1, 2, 3 ] < [ 1, 2, 3, 5 ];  # <list1>[4] is unbound and thus < 5
true
gap> [ 1, , 3, 4 ] < [ 1, -1, 3 ];  # <list1>[2] is unbound and thus < -1
true

Note that for comparing two lists with < or <=, the (relevant) list elements must be comparable with <, which is usually not the case for objects in different families, see 13.1. Also for the possibility to compare lists with other objects, see 13.1.
